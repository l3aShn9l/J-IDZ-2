Особенности работы:
Входные данные - путь до директории. При некорректных данных будет выброшен Exception.
Если существуют зацикливания или некорректный require, то будет выброшен Exception с описанием ошибки, в первую очередь вы получите сообщение о некорректном require, если таковых нет, то пройдет проверка на зацикливания. Таким образом, если исходные данные имеют две ошибки сразу, то вы получите сообщение только о некорректном require.
Многострочными комментариями закомментированы строки кода необходимые для отладки: Вывод матрицы зависимостей, упорядоченного списка файлов, а вывод иллюстрирующий процесс поиска циклов и упорядочивания.
Поскольку реальна ситуация, что может быть несколько файлов верхнего уровня (от которых никто не зависит), то происходит сборка каждого файла. Если же гарантируется, что такой файл один и он вам необходим, то вы можете раскомментировать вывод упорядоченного списка файлов, необходимый вам файл будет самым нижним в выведенном списке.
В условии к заданию было сказано, что необходимо, чтобы были список файлов, упорядоченный по имени, и список файлов, упорядоченный по уровню зависимостей. Оба они имеются и имеют свое значение в ходе исполнения программы.
